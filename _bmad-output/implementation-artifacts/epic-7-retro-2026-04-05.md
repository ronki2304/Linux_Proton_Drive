# Epic 7 Retrospective — End-to-End Test Suite
Date: 2026-04-05 (partial — story 7-3 not yet started)

## Outcome

**2/3 stories done.** Binary e2e tests (7-1) and live integration tests (7-2) shipped. CI workflow (7-3) still backlog.

---

## What Went Well

- **E2e test isolation is solid (7-1).** `XDG_DATA_HOME` override per test, binary pre-flight skip when binary absent, `spawnSync` + stdout/stderr assertion pattern — clean and reproducible.
- **SDK initialization worked (7-2).** `createLiveDriveClient` factory with `buildHttpClient`, `buildAccount` (SRP key derivation), `OpenPGPCryptoWithCryptoProxy` wired up. The full encrypted upload→download cycle works end-to-end.
- **Code review process caught a critical AC4 bug before it could be missed.** The `controlledClient` object spread issue (prototype methods not copied → uploadFile undefined → TypeError in SyncEngine → conflicts never recorded) would have been impossible to catch without a real credentials test run. The review layer caught it.

## What Didn't Go Well

- **Story 7-2 was the most complex story in the project.** Implementing the real SDK initialization required: an OpenPGP crypto proxy adapter (`openpgp-proxy.ts`), a Proton account service (`account-service.ts`), an SRP module adapter (`srp-module.ts`), and full implementations of `uploadFile`, `downloadFile`, `listFolder`, `getFileMetadata`. This complexity was the direct consequence of the 4-story deferral from Epic 3. The accumulated stub debt was large.
- **Object spread on class instance (controlledClient) is a JavaScript footgun.** `{ ...driveClient, listFolder: override }` copies only own enumerable properties — prototype methods are absent on the resulting object. This is a subtle JS semantics issue that TypeScript's type system does not prevent (cast hid it). Lesson: never spread class instances to create mocks — use `Object.create()`.
- **`/integration-tests/` parent folder accumulated across test runs.** A minor AC5 violation — cleaned up by changing to a flat remote path in the code review fix.
- **Pre-existing `cli.test.ts` binary compilation failure** (`spawnSync("bun", ...)` with bun not in PATH) has carried through all epics unresolved. Not introduced by Epic 7 but now the only failing test.

## Key Lessons

1. **Deferred SDK stubs accumulate compound debt.** A 4-story deferral from Epic 3 became the single largest implementation effort in the project. Future stub deferrals should have a hard deadline story in the same epic, not punted to a later one.
2. **Never spread class instances to create test doubles.** `{ ...instance }` omits prototype methods. Use `Object.create(instance)` with method overrides, or a proper wrapper class.
3. **Integration tests without real credentials must be verified by someone who has them.** The `test.skipIf(SKIP)` pattern is correct for CI, but it means code quality of integration tests depends entirely on human + review-layer scrutiny. There is no automated validation loop.
