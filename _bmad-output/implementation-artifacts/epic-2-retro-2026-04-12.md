## Challenges

1. **`better-sqlite3` native addon** — requires C toolchain not present in dev environment; SQLite unit tests cannot run locally; quality risk for Epic 3 where `change_queue` is the backbone of offline resilience
2. **Architecture doc path drift** — Story 2-10 discovered the documented engine path (`/app/lib/protondrive/engine.js`) was based on a bundling assumption that never happened; actual path is `/app/lib/protondrive-engine/dist/src/main.js`; corrected in implementation but not yet in docs
3. **IPC non-dict payload guard — recurring blindspot** — missed in initial implementation in Stories 2-3, 2-6, and 2-7; caught by code review each time; a pattern problem, not a one-off
4. **Auth testing is exhausting manual work** — CAPTCHA + 2FA + token expiry means Jeremy is the single point of verification for the critical auth-to-sync path; no automation possible until Epic 7

## Key Insights

1. **The Epic 1 planning investment is still paying dividends** — no design debates during implementation; clear acceptance criteria on every story; the PRD, architecture, and UX spec are load-bearing artifacts
2. **Story 0 is the correct pattern** — technical debt addressed before feature work prevents cascading issues; confirmed for Epic 3
3. **The IPC payload guard is a team blindspot** — `isinstance(payload, dict)` before `.get()` should be an explicit rule in `project-context.md`, not an implicit expectation caught at review
4. **Live testing surfaces what unit tests can't** — bcrypt slice and Flatpak node path were both invisible to the test suite; only live smoke testing caught them
5. **Manual auth testing debt is accumulating** — sustainable now, growing risk each epic; no automated solution until Epic 7
