# Epic 6 Retrospective — Packaging & Distribution
Date: 2026-04-05

## Outcome

**3/3 stories done (100%).** GitHub Actions CI/release pipeline, Nix flake, AUR PKGBUILD — all shipped.

---

## What Went Well

- **CI pipeline design is solid.** Two-workflow split (PR gate vs. release) is clean. `bun test` + `tsc --noEmit` on every PR, binary build + AppImage + GitHub Release on `v*` tag push.
- **AppImage packaging works.** `appimagetool` integration, AppDir structure, binary embedding — functional and tested.
- **AUR PKGBUILD uses pre-built binary (Option B).** No `bun` makedepend for end users — correct choice for a distributed CLI tool. PKGBUILD + .SRCINFO both present.

## What Didn't Go Well

- **GitHub Actions pinned to mutable version tags.** `setup-bun@v2` and `action-gh-release@v2` used mutable tags — these can be force-pushed for supply-chain injection. Caught in review (6-1). Should have been pinned to commit SHAs from the start.
- **Nix `bun install --frozen-lockfile` runs in network-isolated sandbox.** Nix derivations can't make network calls — `bun install` always fails in that context. Required a fixed-output derivation or vendored deps approach. Caught in review (6-2). A fundamental Nix sandbox constraint that should have been known upfront.
- **AUR `sha256sums=SKIP`** — valid as a pre-release placeholder but AUR guidelines prohibit SKIP for binary sources. Strengthened comment to require real hashes before submission.
- **`flake.lock` not generated** — Nix is not installed in the dev environment. `nix flake lock` must be run before first release. Manual step documented but not automated.

## Key Lessons

1. **Always pin GitHub Actions to commit SHAs, not tags.** Tags are mutable. This is a supply-chain security requirement, not a preference.
2. **Nix derivation networking rules must be understood before designing the build.** `bun install` in a Nix sandbox is a category error. Research the fetch model (fixed-output derivation, `npmDeps`, or vendored node_modules) before writing the flake.
3. **Packaging stories need more research spikes.** All three packaging stories (AppImage, Nix, AUR) had packaging-system-specific constraints that weren't in the spec. Consider a research task at the start of packaging epics.
